PARTE 1 — Preparar tu Fedora (hoy)

1.1 Verificar SSH

Abrí la terminal en Fedora:

ssh -V

Si no está instalado:

sudo dnf install -y openssh-clients

1.2 Qué necesitás del panel DonWeb

  • Usuario root y contraseña (al activar el VPS).
  • VPS en estado Activo / Running.
  • IP 200.58.99.253 accesible.

Panel: cuenta DonWeb → Cloud Servers / VPS → seleccionar el servidor. Ahí ves IP, reinicio y consola de emergencia si hace falta.

1.3 Primer ingreso con contraseña (mañana)

ssh root@200.58.99.253

Si pregunta Are you sure you want to continue connecting? escribí yes y Enter.

Ingresá la contraseña de root (no se ve al tipear). Alternativa:

ssh root@vps-6055529-x.dattaweb.com

1.4 (Opcional hoy) Crear clave SSH en Fedora

Recomendado para no depender solo de contraseña. La copiás al servidor mañana, tras crear el usuario erp.

ssh-keygen -t ed25519 -C "hernan-fedora-erp"
# Enter en ruta por defecto (~/.ssh/id_ed25519)
# Passphrase opcional

# Mañana, tras crear usuario erp en el VPS:
ssh-copy-id erp@200.58.99.253

PARTE 2 — Configuración inicial del VPS (como root)

DonWeb suele entregar Ubuntu. Ejecutá en orden la primera vez.

2.1 Actualizar el sistema

apt update && apt upgrade -y

2.2 Crear usuario de trabajo

adduser erp
usermod -aG sudo erp

Probá desde otra terminal en Fedora (sin cerrar root):

ssh erp@200.58.99.253

2.3 Firewall (solo 22, 80, 443)

sudo apt install -y ufw
sudo ufw default deny incoming
sudo ufw default allow outgoing
sudo ufw allow OpenSSH
sudo ufw allow 80/tcp
sudo ufw allow 443/tcp
sudo ufw enable
sudo ufw status

No abras 5432 (PostgreSQL) ni 3000 (API) al exterior.

2.4 Instalar Docker

sudo apt install -y ca-certificates curl
sudo install -m 0755 -d /etc/apt/keyrings
sudo curl -fsSL https://download.docker.com/linux/ubuntu/gpg -o /etc/apt/keyrings/docker.asc
sudo chmod a+r /etc/apt/keyrings/docker.asc

echo "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.asc] https://download.docker.com/linux/ubuntu $(. /etc/os-release && echo $VERSION_CODENAME) stable" | sudo tee /etc/apt/sources.list.d/docker.list > /dev/null

sudo apt update
sudo apt install -y docker-ce docker-ce-cli containerd.io docker-compose-plugin
sudo usermod -aG docker erp

Cerrá sesión y volvé a entrar como erp para usar Docker sin sudo.

2.5 Instalar Node.js 20

curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt install -y nodejs
node -v
npm -v

2.6 PM2, nginx, Certbot y Git

sudo npm install -g pm2
sudo apt install -y nginx certbot python3-certbot-nginx git

PARTE 3 — Clonar el proyecto

cd /opt
sudo mkdir -p /opt/ERPTienda
sudo chown erp:erp /opt/ERPTienda
git clone https://github.com/hernanXDD/ERPTienda.git
cd ERPTienda

Si el repo es privado, GitHub pide usuario y Personal Access Token (no la contraseña de GitHub).

PARTE 4 — PostgreSQL en Docker

4.1 Variables

cp .env.production.example .env.production
nano .env.production
  • POSTGRES_USER=erp
  • POSTGRES_PASSWORD=contraseña-fuerte
  • POSTGRES_DB=ERPTienda

Guardar en nano: Ctrl+O, Enter, Ctrl+X.

4.2 Levantar la base

docker compose -f docker-compose.prod.yml --env-file .env.production up -d
docker compose -f docker-compose.prod.yml ps

PostgreSQL queda en 127.0.0.1:5432 (solo el VPS).

PARTE 6 — nginx + HTTPS

6.1 nginx

sudo cp deploy/nginx/erp-tienda-api.conf.example /etc/nginx/sites-available/erp-tienda-api
sudo ln -s /etc/nginx/sites-available/erp-tienda-api /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl reload nginx

6.2 Certificado SSL

sudo certbot --nginx -d vps-6055529-x.dattaweb.com

6.3 Probar desde Fedora

curl -s https://vps-6055529-x.dattaweb.com/api/salud

PARTE 7 — Frontend en Vercel

  1. Entrá a vercel.com con GitHub.
  2. Import Project → repo hernanXDD/ERPTienda.
  3. Root Directory: frontend
  4. Build: npm run build — Output: dist
  5. Variable Production: VITE_API_BASE_URL=https://vps-6055529-x.dattaweb.com/api
  6. Deploy.
  7. Copiá la URL de Vercel (ej. https://erp-tienda-xxx.vercel.app).
  8. En el VPS: nano backend/.env → CORS_ORIGENES = esa URL exacta.
  9. pm2 reload erp-tienda-api
