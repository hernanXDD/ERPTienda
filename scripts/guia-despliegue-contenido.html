3. Preparar SSH en Fedora (tu PC)

3.1 Cliente SSH

ssh -V
sudo dnf install -y openssh-clients

3.2 Crear clave (primera vez)

ssh-keygen -t ed25519 -C "hernan-fedora-erp"
cat ~/.ssh/id_ed25519.pub
ArchivoUso
~/.ssh/id_ed25519Privada — no compartir
~/.ssh/id_ed25519.pubPública — va al VPS

3.3 Atajo ~/.ssh/config (opcional)

Host erp-vps
    HostName 200.58.99.253
    User erp
    IdentityFile ~/.ssh/id_ed25519
chmod 600 ~/.ssh/config

4. Primer ingreso como root

No cierres esta sesión root hasta confirmar que podés entrar con erp y clave SSH (sección 6).

5. Configuración inicial del servidor

Ejecutar en el VPS como root.

5.1 Actualizar sistema

apt update && apt upgrade -y

5.2 Crear usuario erp con sudo

adduser erp
usermod -aG sudo erp

Elegí contraseña para erp (distinta a root). Verificar:

su - erp
sudo whoami
exit

Debe imprimir root.

5.3 Paquetes básicos

apt install -y curl git ufw fail2ban

5.4 Zona horaria Argentina

timedatectl set-timezone America/Argentina/Buenos_Aires
timedatectl

5.5 Firewall UFW

ufw default deny incoming
ufw default allow outgoing
ufw allow OpenSSH
ufw allow 80/tcp
ufw allow 443/tcp
ufw enable
ufw status
No abrir 5432 (Postgres) ni 3000 al exterior. Easypanel usa la red interna Docker.

5.6 Fail2ban (protección SSH)

systemctl enable fail2ban
systemctl start fail2ban

6. Copiar clave SSH al usuario erp

Desde Fedora (mantener sesión root abierta por las dudas):

ssh-copy-id erp@200.58.99.253

Alternativa manual:

cat ~/.ssh/id_ed25519.pub | ssh erp@200.58.99.253 \
  "mkdir -p ~/.ssh && chmod 700 ~/.ssh && cat >> ~/.ssh/authorized_keys && chmod 600 ~/.ssh/authorized_keys"

Probar en terminal nueva:

ssh erp@200.58.99.253
# o: ssh erp-vps

Debe entrar con clave sin pedir contraseña de erp.

7. Deshabilitar root por SSH

Solo después de confirmar que ssh erp@... funciona con clave.
sudo cp /etc/ssh/sshd_config /etc/ssh/sshd_config.bak
sudo nano /etc/ssh/sshd_config

Dejar así (descomentar o agregar):

PermitRootLogin no
PasswordAuthentication no
PubkeyAuthentication yes
sudo systemctl restart ssh

Probar desde Fedora:

ssh root@200.58.99.253    # debe FALLAR (correcto)
ssh erp@200.58.99.253     # debe ENTRAR con clave (correcto)

Si erp no entra, restaurar con la sesión que sigue abierta:

sudo cp /etc/ssh/sshd_config.bak /etc/ssh/sshd_config
sudo systemctl restart ssh
De ahora en más: ssh erp@200.58.99.253. Root solo por consola de emergencia en DonWeb.

8. Easypanel — verificar o instalar

Si DonWeb ya instaló Easypanel, abrí su URL en el navegador.

Si no está instalado (como erp con sudo):

sudo apt install -y docker.io docker-compose-plugin
sudo usermod -aG docker erp
# cerrar sesión SSH y volver a entrar como erp

curl -fsSL https://get.easypanel.io | sudo bash
