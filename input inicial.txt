# PROYECTO ERP - TIENDA DE ROPA FEMENINA
# DOCUMENTO MAESTRO PARA CURSOR

================================================================================
OBJETIVO GENERAL
================================================================================

Quiero crear un sistema web profesional para gestión diaria de una tienda de ropa femenina.

El proyecto debe construirse con arquitectura moderna, escalable, segura y mantenible.

El sistema debe estar preparado para crecer en el futuro y mantener estándares enterprise desde el inicio.

NO quiero desarrollar todo junto.
Quiero avanzar módulo por módulo y esperar aprobación antes de continuar.

================================================================================
ESTRUCTURA GENERAL DEL PROYECTO
================================================================================

El proyecto debe separarse obligatoriamente en dos carpetas principales:

/backend
/frontend

NO mezclar frontend y backend.

Cada uno debe tener:
- su propia configuración
- su propio package.json
- su propia estructura
- sus propias dependencias
- sus propias variables de entorno

Estructura general esperada:

/proyecto
│
├── /backend
│
├── /frontend
│
├── docker-compose.yml
├── .gitignore
├── README.md
└── docs/

ETAPA INICIAL DEL DESARROLLO
================================================================================

IMPORTANTE:

En la primera etapa NO trabajaremos directamente contra el backend real.

Primero quiero desarrollar el frontend utilizando:
- archivos JSON locales
- mocks
- datos simulados
- estructuras temporales

Objetivos de esta etapa:
- acelerar el desarrollo visual
- validar pantallas
- validar flujos
- validar UX/UI
- validar estructura de datos
- validar relaciones entre módulos

El frontend debe consumir inicialmente:
- archivos JSON
- servicios mockeados
- stores simulados

Luego, en una segunda etapa:
- reemplazaremos mocks por backend real
- conectaremos NestJS + Prisma + PostgreSQL
- mantendremos exactamente la misma estructura de datos

Por lo tanto:
- diseñar desde el inicio pensando en futura integración real
- mantener tipados correctos
- mantener interfaces reutilizables
- evitar acoplar el frontend a mocks temporales
