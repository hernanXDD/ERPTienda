================================================================================
ERP TIENDA — Comandos para levantar y probar el sistema (local)
================================================================================

Requisitos: Node.js 20+, Docker, npm.

Rutas: ejecutar desde la raíz del proyecto (donde está docker-compose.yml).


--------------------------------------------------------------------------------
1) CONFIGURACIÓN INICIAL / REACTIVAR DESARROLLO LOCAL
--------------------------------------------------------------------------------

cd /ruta/al/proyecto/ERPTienda

# Un solo comando: .env, Docker, dependencias, migraciones y semilla:
chmod +x scripts/preparar-desarrollo-local.sh dev.sh
./scripts/preparar-desarrollo-local.sh

# Fedora: si 5432 está ocupado por PostgreSQL del sistema, .env ya usa 5433.
# Verificar que backend/.env tenga el mismo puerto en DATABASE_URL.

# Levantar todo (PostgreSQL + API + frontend):
./dev.sh


--------------------------------------------------------------------------------
2) BASE DE DATOS (Docker)
--------------------------------------------------------------------------------

cd /home/hernan/Proyectos/ERPTienda
docker compose up -d
docker compose ps

# Ver logs de PostgreSQL (opcional):
# docker compose logs -f postgres


--------------------------------------------------------------------------------
3) BACKEND — migraciones, semilla y API
--------------------------------------------------------------------------------

cd /home/hernan/Proyectos/ERPTienda/backend

npx prisma generate
npm run db:migrate
npm run db:seed

# Desarrollo (recarga automática al guardar código):
npm run start:dev

# Producción local (compilar y ejecutar):
# npm run build
# npm run start:prod

# Comprobar que la API responde:
# http://localhost:3000/api/salud


--------------------------------------------------------------------------------
4) FRONTEND — interfaz web
--------------------------------------------------------------------------------

cd /home/hernan/Proyectos/ERPTienda/frontend

npm run dev

# Abrir en el navegador:
# http://localhost:5173
# (si el puerto está ocupado, Vite puede usar 5174 u otro — mirar la consola)

# El frontend usa proxy Vite (/api → http://localhost:3000). No requiere .env en desarrollo local.


--------------------------------------------------------------------------------
5) LEVANTAR TODO
--------------------------------------------------------------------------------

Opción A — un comando (recomendado):
  cd /home/hernan/Proyectos/ERPTienda
  ./dev.sh

Opción B — 3 terminales:
  Terminal 1: docker compose up -d
  Terminal 2: cd backend && npm run start:dev
  Terminal 3: cd frontend && npm run dev


--------------------------------------------------------------------------------
6) LOGIN INICIAL (después de npm run db:seed)
--------------------------------------------------------------------------------

Usuario          Contraseña
-------          ----------
admin            12345678

Cambiar la contraseña del administrador antes de probar en red compartida.


--------------------------------------------------------------------------------
7) PRIMERA MIGRACIÓN (solo si la base está vacía y no hay migraciones aplicadas)
--------------------------------------------------------------------------------

cd /home/hernan/Proyectos/ERPTienda/backend
npx prisma migrate dev --name inicial
npm run db:seed


--------------------------------------------------------------------------------
8) REINICIAR BASE DESDE CERO (como primera migración — borra todos los datos)
--------------------------------------------------------------------------------

cd /home/hernan/Proyectos/ERPTienda
chmod +x scripts/reiniciar-base-local.sh
./scripts/reiniciar-base-local.sh

# Equivalente manual:
#   docker compose down -v
#   docker compose up -d
#   cd backend && npx prisma generate && npm run db:migrate && npm run db:seed


--------------------------------------------------------------------------------
9) DETENER SERVICIOS
--------------------------------------------------------------------------------

# Backend / Frontend: Ctrl+C en cada terminal

# PostgreSQL (Docker):
cd /home/hernan/Proyectos/ERPTienda
docker compose down


--------------------------------------------------------------------------------
10) COMANDOS ÚTILES
--------------------------------------------------------------------------------

# Prisma Studio (explorar datos en el navegador):
cd backend
npx prisma studio

# Build de verificación (frontend):
cd frontend
npm run build

# Build de verificación (backend):
cd backend
npm run build

# Estado de migraciones:
cd backend
npx prisma migrate status
